謎解き脱出ブース - 使い方
====================================

■ ファイル構成
  index.html   … 画面のHTML（4画面すべてここに入っています）
  style.css    … イカゲーム風の配色・デザイン
  script.js    … 画面遷移・タイマー・キーパッド・正誤判定のロジック
  no1/ ～ no5/ … 謎解き問題フォルダ（各フォルダに question.jpg と answer.txt）

  コードは分割していますが、ファイル名は上記の通り固定です。
  index.html が style.css と script.js を相対パスで読み込んでいるので、
  3つのファイルは必ず同じフォルダに置いてください。

■ 問題の入れ替え方
  no1〜no5 の中の question.jpg（問題の画像）と answer.txt（正解の文字列）
  を差し替えるだけで問題を変更できます。
  ※ 今入っているものはサンプル（ダミー）の問題です。本番の問題画像・
    正解文字に置き換えてください。
  ※ answer.txt は文字コード UTF-8、余計な改行や空白が入らないように
    保存してください（スクリプト側で前後の空白は自動的に取り除きます）。
  ※ フォルダを増やしたい場合は script.js の先頭にある
       const FOLDERS = ["no1", "no2", "no3", "no4", "no5"];
    に追加してください。

■ 動作させる方法（重要）
  ブラウザのセキュリティ制限により、index.html をダブルクリックして
  file:// で直接開くと画像・answer.txt の読み込みに失敗する場合があります。
  必ず簡易Webサーバー経由で開いてください。

  例）このフォルダの中で下記コマンドを実行し、
      表示されたURL（例: http://localhost:8000）をタブレットのブラウザで開く。

      Python がある場合:
        python3 -m http.server 8000

      Node.js がある場合（npxが使える場合）:
        npx http-server -p 8000

  同じWi-Fiに接続したタブレットから、上記PCのIPアドレスを使って
  http://（PCのIPアドレス）:8000 のように開けば、タブレット単体でも
  ブースとして常設できます。もしくは、上記コマンドをタブレット本体
  （Android等でTermux等が使える場合）で直接動かしても構いません。

■ 画面の流れ（仕様どおり）
  1. 待機画面：「1人」「2人」「3人」ボタン
  2. 問題画面：人数に応じた制限時間（90/75/60秒）をセットしてスタート。
     左に問題画像、右上に解答入力欄、右下にひらがなキーパッド（5列）。
     不正解時は×を2秒間表示（タイマーは止まらない）。
     時間切れで自動的に「脱落画面」へ。
  3. 正解画面：赤背景・白文字で「正解 次の部屋に進め」を5秒間表示 → 待機画面へ。
  4. 脱落画面：青背景・白文字で「脱落 直ちに退出せよ」を5秒間表示 → 待機画面へ。

■ タブレットで使う際のヒント
  ・ブラウザを全画面（キオスク）モードで開くと没入感が高まります。
  ・自動スリープをオフにしておくと、待機画面のまま消灯しません。
  ・誤タップでブラウザの戻る操作等が起きないよう、キオスクブラウザアプリの
    利用もおすすめです。
